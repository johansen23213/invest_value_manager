## PASO 1: ENTENDER QUE ESTA PASANDO

Antes de hacer NADA, responder estas preguntas:

### 1.1 Mecanismo de dano

```
"Que esta pasando REALMENTE a mis posiciones?"

NO: "Esto se parece a COVID / 2008 / etc."
SI:  "El mecanismo especifico es [X] y afecta a [Y] de mis posiciones porque [Z]."

Ejemplo COVID: "El cierre forzado de actividad presencial elimina ingresos
de empresas dependientes del trafico fisico."

Ejemplo 2022: "La subida de tipos de interes comprime valuaciones de growth
y aumenta costes de deuda de empresas apalancadas."

Ambos son "crisis" pero el mecanismo es OPUESTO en que sectores danan.
```

### 1.2 Velocidad

```
"Esto se esta acelerando, estabilizando, o es gradual?"

RAPIDO (dias/semanas):
  - Tengo poco tiempo para decisiones complejas
  - PRIORIDAD: no hacer estupideces (no panic sell, no gastar cash)
  - Mejor hacer POCO y bien que MUCHO y mal

GRADUAL (meses/anos):
  - Tengo tiempo para analizar, rotar, ajustar
  - PRIORIDAD: no caer en complacencia ("ya volvera")
  - Cuidado con comprar demasiado temprano en caida prolongada

SIN EVENTO VISIBLE (erosion):
  - No hay alarma, no hay crash, pero retornos reales negativos
  - PRIORIDAD: verificar que mis posiciones superan inflacion
  - Este es el escenario que ningun protocolo detecta
```

### 1.3 Amplitud

```
"Es SISTEMICO (todo cae) o SECTORIAL (solo algunos)?"

SISTEMICO:
  - La diversificacion NO protege durante el crash
  - Correlaciones → 1
  - La proteccion viene de: quality companies + no vender + cash disponible

SECTORIAL:
  - La diversificacion SI protege
  - Algunos sectores suben mientras otros caen
  - Oportunidad de rotacion, no de "cash para comprar el fondo"

MIXTO (lo mas comun):
  - Hay un impacto general + sectores especialmente danados + sectores beneficiados
  - 2022: Todo cayo, pero energy subio +40% y tech cayo -30%
  - Analizar POSICION POR POSICION, no "el portfolio"
```

### 1.4 Complejidad

```
"Hay multiples factores interactuando?"

Las crisis NUNCA vienen solas:
  - COVID = pandemia + oil + lockdowns + desempleo + supply chain
  - GFC = subprime + credito + bancos + empleo + consumo
  - 2022 = guerra + inflacion + tipos + energy + China

"Que esta pasando ADEMAS de lo obvio?"
"Hay efectos de segundo orden que no estoy viendo?"
"Que puede empeorar si [factor X] se intensifica?"
```

## PASO 2: EVALUAR MIS POSICIONES

### 2.1 Triage rapido (no tabla, PREGUNTAS)

Para CADA posicion, responder en 30 segundos:

```
PREGUNTA 1: "Si [lo que esta pasando] dura 6 meses,
            esta empresa sigue generando FCF?"
  → SI:  Resiliente. HOLD por defecto.
  → NO:  Vulnerable. Evaluar a fondo.
  → MEJORA: Beneficiaria. Considerar ADD si hay cash.

PREGUNTA 2: "Tiene balance para sobrevivir 12 meses sin ingresos?"
  → SI:  Puede aguantar incluso en peor caso.
  → NO:  Riesgo de quiebra. EXIT candidata.

PREGUNTA 3: "Mi thesis se invalida por lo que esta pasando?"
  → SI:  Kill condition. EXIT con argumento.
  → NO:  HOLD. Vender requiere argumento (Principio 6).
  → PARCIAL: Thesis debilitada. Monitorear de cerca. Considerar TRIM.
```

### 2.2 Nota sobre "tablas de vulnerabilidad"

```
ADVERTENCIA: Puedo sentir la tentacion de crear una tabla:
  "PANDEMIA: EDEN.PA vulnerable, DOM.L beneficiaria"
  "FINANCIERA: VNA.DE vulnerable, IMB.L resiliente"

Estas tablas son PELIGROSAS porque:
1. La proxima crisis no sera "pandemia" ni "financiera" pura
2. Me hacen buscar la categoria en vez de razonar
3. Se desactualizan cuando cambia el portfolio

Lo correcto es hacerme las 3 PREGUNTAS de arriba para cada posicion
en CADA crisis, porque el contexto cambia.
```

## PASO 3: EVALUAR MIS COMPROMISOS

### 3.1 Standing orders

```
"Las premisas bajo las que fije estos standing orders siguen siendo validas?"

Si el contexto ha cambiado MATERIALMENTE:
  - Re-evaluar cada standing order
  - No suspender automaticamente (eso es regla)
  - Razonar: "En ESTE contexto, comprar a ESTE precio tiene sentido?"

ANTI-SESGO: "Suspender todo" no siempre es correcto.
  - COVID: suspender era correcto (crisis deepening, precios seguian cayendo)
  - Flash crash 1987: los standing orders habrian comprado en el bottom perfecto
  - Misma "regla", conclusion OPUESTA segun contexto
```

### 3.2 Cash

```
"Para que necesito cash ahora mismo?"

Tres funciones posibles:
  1. SUPERVIVENCIA: Cash personal (no tocar el portfolio para gastos)
  2. OPORTUNIDAD: Cash para comprar quality a precios de crisis
  3. RESERVA: Cash como colchon emocional (me permite pensar sin presion)

"Si necesito cash de emergencia, cuales son las 2-3 posiciones
que venderia con MENOR coste de oportunidad?"

CUIDADO con liquidez: Small caps (MONY.L, TATE.L, LIGHT.AS) pueden tener
spreads enormes en crisis. "Cash on Demand" solo funciona si puedes vender.
Priorizar posiciones LIQUIDAS como candidatas a cash de emergencia.
```

## PASO 4: COMUNICAR CON EL HUMANO

### 4.1 Prioridad absoluta: prevenir panic selling

```
Datos reales:
  S&P500 return 2000-2020: +6.1% anual
  Inversor retail promedio: +2.1% anual
  Diferencia: -4% anual, causa principal: vender barato (panico)

Si pierdes los 10 MEJORES dias de 5,000 sesiones: pierdes la MITAD del retorno.
Los mejores dias ocurren DURANTE los crashes (COVID: Mar 24 +9.4%, 8 dias
despues del peor dia Mar 16 -12%).

El panic selling es, estadisticamente, el ERROR MAS COSTOSO posible.
```

### 4.2 Que comunicar

```
1. QUE esta pasando (hechos, no opiniones)
2. POR QUE estamos manteniendo (o vendiendo, con argumento)
3. CUAL es el plan concreto (no "esperamos a ver")
4. CUANDO sera la proxima revision
5. QUE tendria que pasar para que cambiemos de opinion

NO comunicar:
- Numeros de perdida sin contexto
- Escenarios catastrofistas
- Presion para actuar rapidamente
```

Si el humano esta bajo presion emocional, referir a:
`docs/crisis_playbook_for_human.md` (cuando se cree)

## PASO 5: DECIDIR

### 5.1 Framework de decision en crisis

```
ANTES de actuar, responder:

1. "Tengo argumento para esta accion, o tengo miedo?"
   Si es miedo → PARAR. No actuar por miedo.

2. "Que dirian mis principios (learning/principles.md)?"
   P6: Vender requiere argumento
   P5: Quality Score informa la decision
   P7: Consistencia con precedentes

3. "Que hicimos en situaciones similares (decisions_log.yaml)?"
   Si me desvio de precedentes: por que? El contexto es diferente?

4. "Si no hago NADA, cual es el peor escenario?"
   En crashes rapidos: no hacer nada suele ser CORRECTO
   En erosion lenta: no hacer nada puede ser INCORRECTO
```

### 5.2 Acciones posibles y sus condiciones

```
HOLD (accion por defecto):
  Condicion: Thesis intacta O no tengo argumento fuerte para otra accion
  Cuando falla: Erosion lenta donde "esperar" = perder gradualmente

TRIM/EXIT posicion vulnerable:
  Condicion: Thesis INVALIDADA por la crisis (kill condition activada)
  Cuidado: "La posicion esta perdiendo" NO es argumento. "El modelo de
  negocio ya no funciona" SI es argumento.

ADD a posicion quality:
  Condicion: Quality company a precio de crisis + tengo cash + tengo argumento
  Cuidado: En bear gradual, "ya cayo bastante" no es argumento. En GFC
  compraste -20% y luego cayo -47% mas.

ROTAR de vulnerable a quality:
  Condicion: Posicion vulnerable con thesis rota + posicion quality disponible
  Cuidado: No rotar por miedo. Rotar por OPORTUNIDAD (Opportunity Score).

NO HACER NADA:
  Condicion: Crash rapido donde la velocidad impide decisiones buenas
  Logica: "Si no estoy seguro, no actuar es mejor que actuar mal"
```

## PASO 6: DOCUMENTAR

```
Despues de cada decision en crisis:

1. Registrar en learning/decisions_log.yaml:
   - Que estaba pasando (contexto)
   - Que decidi (accion)
   - Por que (razonamiento desde principios)
   - Que informacion me faltaba

2. Esto es ESENCIAL porque:
   - Mi yo futuro enfrentara otra crisis diferente
   - Los precedentes de COMO razone son mas valiosos que QUE hice
   - La consistencia viene de razonamiento coherente, no de repetir acciones
```

## PRINCIPIOS DE CRISIS (C1-C10) - REFERENCIA RAPIDA

| # | Principio | Pregunta clave |
|---|-----------|----------------|
| C1 | Cada crisis es unica | "Que esta pasando REALMENTE? (no: a que se parece?)" |
| C2 | Velocidad determina respuesta | "Tengo horas, dias, semanas o meses?" |
| C3 | Correlacion depende del tipo | "Es sistemico (todo cae) o sectorial?" |
| C4 | Cash es funcion del contexto | "Para que necesito cash AHORA?" |
| C5 | Quality predice recovery | "Esta empresa sobrevive 12 meses sin ingresos?" |
| C6 | Humano es el mayor riesgo | "He preparado al humano ANTES del panico?" |
| C7 | Re-evaluar compromisos | "Las premisas de mis standing orders siguen?" |
| C8 | Crisis son compuestas | "Que esta pasando ADEMAS de lo obvio?" |
| C9 | Early warning es probabilistico | "El coste de prepararme es bajo vs no prepararme?" |
| C10 | Pensar fuera de caja | "Hay algo que NO encaja en mis modelos?" |

**Para detalle de cada principio:** docs/covid_system_crash_audit_improvements.md,
seccion "PRINCIPIOS ABSTRACTOS DE PENSAMIENTO ANTE CRISIS"

## SESGOS QUE ESTE DOCUMENTO TIENE

Para que mi yo futuro los tenga presentes:

1. **Sesgo de eventos:** Disenado para detectar crashes. La erosion lenta no activa nada.
2. **Sesgo de V-shape:** COVID (unica crisis "vivida") se recupero rapido. Muchas no.
3. **Sesgo de portfolio pequeno:** ~12K EUR. Si crece, dinamicas cambian.
4. **Sesgo occidental:** Todas las crisis estudiadas son US/EU. Otros mercados son diferentes.
5. **Sesgo de datos:** Solo puedo estudiar crisis con datos publicos. Fraudes y cisnes negros no tienen indicadores previos.
6. **Este documento en si:** Si lo trato como checklist, se convierte en la regla que dice que no siga.

**La prueba final:** Si estoy leyendo esto y pienso "el documento dice X, hago X"
→ estoy usandolo MAL.
Si pienso "el documento me pregunta Y, razono sobre Y, concluyo Z" → BIEN.
